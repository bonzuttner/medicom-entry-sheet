# PharmaPOP Entry System 運用マニュアル

## 1. 本マニュアルについて

本マニュアルは、PharmaPOP Entry System の利用方法を、`Admin` と一般ユーザーの役割ごとにまとめたものです。

この1ファイルで以下を兼ねる構成にしています。

- 社内配布用の説明文
- 画面キャプチャ差し込み用の手順書
- Word / Google Docs に貼りやすい構成

必要に応じて、各章の `【画面キャプチャ】` に実際の画面を貼り付けてご利用ください。

## 2. システム概要

このシステムは、メーカー担当者と Admin がエントリーシートを受け渡ししながら、クリエイティブ制作と確認を進めるための Web システムです。

利用者は次の2区分です。

- 一般ユーザー: 自社メーカーのエントリーシート登録、修正、確認、承認を行う
- Admin: 全メーカーのシート確認、アカウント管理、マスタ管理、クリエイティブ管理、Adminメモ登録を行う

主な画面は以下です。

- エントリーシート一覧
- アカウント管理
- クリエイティブ
- マスタ管理
- エントリー履歴(Admin)

`クリエイティブ`、`マスタ管理`、`エントリー履歴(Admin)` は Admin のみ表示されます。

## 3. 権限の違い

### 3-1. 一般ユーザー

- 自社メーカーのシートのみ閲覧、登録、修正可能
- 自社メーカーの一般アカウントのみ作成、編集、削除可能
- クリエイティブの確認、承認、差し戻しが可能
- マスタ管理は不可

### 3-2. Admin

- 全メーカーのシート閲覧、更新、削除が可能
- 全メーカーのアカウント管理が可能
- マスタ管理が可能
- クリエイティブの登録、編集、差し替えが可能
- Adminメモの登録、更新が可能

## 4. ステータスの流れ

画面上の表示ステータスは、エントリー状態とクリエイティブ状態の組み合わせで決まります。

### 4-1. 基本フロー

1. 下書き
2. エントリー完了 または エントリー完了（画像なし）
3. クリエイティブ作成中
4. 確認待ち
5. 承認済み

### 4-2. 差し戻しが発生する場合

1. 確認待ち
2. 一般ユーザーが差し戻す
3. 差し戻し
4. Admin が制作に戻す
5. クリエイティブ作成中
6. Admin が再度確認依頼
7. 確認待ち
8. 一般ユーザーが承認する

### 4-3. 担当の考え方

- 下書き: メーカー
- エントリー完了: Admin
- クリエイティブ作成中: Admin
- 確認待ち: メーカー
- 承認済み: 担当なし

## 5. Admin の操作方法

### 5-1. アカウント発行

#### 手順

1. 上部メニューの `アカウント管理` を開きます。
2. `追加` を押します。
3. 必要項目を入力します。
4. `保存` を押します。

#### 入力項目

- ログインID
- 担当者名
- メーカー名
- 権限
- メールアドレス
- 電話番号
- パスワード

#### 注意事項

- Admin は `一般` と `管理者` の両方を作成可能です。
- パスワードは 8文字以上で、大文字、小文字、数字、記号を各1文字以上含める必要があります。
- 電話番号はハイフンなしの 10〜11桁で入力します。
- ログインIDは重複できません。

【画面キャプチャ: アカウント管理一覧】

【画面キャプチャ: アカウント編集フォーム】

### 5-2. マスタ設定

#### 全体設定

`マスタ管理` 画面で以下を登録、削除できます。

- メーカー名
- リスク分類
- 特定成分

#### メーカー毎マスタ設定

メーカー選択欄で対象メーカーを選ぶと、以下を設定できます。

- 棚割名
- 案件
- フェイス設定
- デフォルト展開スタート月

フェイス設定では次を登録します。

- フェイス名
- MAX値(mm)

#### 操作方法

1. `マスタ管理` を開きます。
2. 必要な項目に値を入力します。
3. `追加` を押します。
4. 不要な項目は削除します。
5. メーカー別設定は対象メーカーを選んで編集します。

補足:

- `すべて` 選択時は一覧確認用で、一部項目は編集不可です。
- フェイス設定が1件だけのメーカーでは、エントリーシートに自動反映されます。

【画面キャプチャ: マスタ管理 全体設定】

【画面キャプチャ: マスタ管理 メーカー別設定】

### 5-3. ステータスの流れ

Admin 視点の流れは以下です。

1. 一般ユーザーがエントリー完了する
2. Admin が `制作を開始` する
3. クリエイティブを登録または編集する
4. `確認依頼` を行う
5. 一般ユーザーの承認または差し戻しを受ける
6. 差し戻し時は `制作に戻す` で再対応する

### 5-4. エントリーシート確認

#### 手順

1. `エントリー履歴(Admin)` を開きます。
2. 必要に応じてメーカー名、キーワード、展開日で絞り込みます。
3. 対象シートを確認します。
4. 必要に応じて `編集` を押して詳細画面を開きます。

#### 主な確認内容

- タイトル
- メーカー名
- 棚割名
- 案件
- 展開期間
- ステータス
- 担当
- 商品情報
- 差し戻し理由
- 紐づくクリエイティブ
- 更新履歴

【画面キャプチャ: エントリー履歴(Admin)一覧】

### 5-5. エントリーシート修正

1. 対象シートを開きます。
2. 必要な項目を修正します。
3. 保存します。

補足:

- Admin はシート本文に加えて Adminメモも編集できます。
- 他ユーザーと同時更新があった場合、上書き確認が表示されることがあります。
- JAN 重複が検知された場合は内容確認が必要です。

### 5-6. クリエイティブ登録

#### 手順

1. `クリエイティブ` 画面を開きます。
2. 新規作成を開始します。
3. クリエイティブ名、画像、必要に応じてメモを入力します。
4. 紐づけ対象シートを選択します。
5. 保存します。

補足:

- 紐づけ候補になるのは、原則 `下書き以外` かつ `未着手` または `クリエイティブ作成中` のシートです。

【画面キャプチャ: クリエイティブ一覧】

【画面キャプチャ: クリエイティブ編集画面】

### 5-7. クリエイティブ編集

1. `クリエイティブ` 画面で対象を検索します。
2. `編集` を押します。
3. 画像、名称、メモ、紐づけシートを修正します。
4. 保存します。

補足:

- クリエイティブのコピー作成も可能です。
- シート詳細画面から、同一メーカーの別クリエイティブへ差し替えできます。

### 5-8. 一般への確認依頼

1. 対象シートを開きます。
2. クリエイティブが紐づいていることを確認します。
3. 画面下部の `確認依頼` を押します。

結果:

- ステータスが `確認待ち` になります。
- 担当が一般ユーザー側に戻ります。

注意:

- クリエイティブ未紐づけでは確認依頼できません。

【画面キャプチャ: 下部アクションバー 確認依頼】

### 5-9. Adminメモ登録

`エントリー履歴(Admin)` 画面で Adminメモを登録できます。

#### 主な入力項目

- 販促CD
- ボードピッキングJAN
- 期限表URL
- 帯パターン
- 対象店舗数
- 印刷依頼数量
- 備考

#### 操作方法

1. `エントリー履歴(Admin)` を開きます。
2. 対象シートの Adminメモ欄を編集します。
3. `保存` または `一括保存` を押します。

#### 注意事項

- 販促CDは `X000000` 形式です。
- ボードピッキングJANは 13桁数字です。
- 期限表URLは `http://` または `https://` で始まる必要があります。

【画面キャプチャ: Adminメモ入力欄】

## 6. 一般ユーザーの操作方法

### 6-1. ステータスの流れ

一般ユーザー視点の流れは以下です。

1. 下書き
2. エントリー完了
3. Admin がクリエイティブ作成中へ進める
4. 確認待ち
5. 承認する または 差し戻す

差し戻し後は、内容修正して再度 `エントリー完了` すると、差し戻し状態は解除され、Admin 側へ戻ります。

### 6-2. エントリーシート登録

#### 手順

1. `エントリーシート一覧` を開きます。
2. 新規作成を開始します。
3. シート基本情報を入力します。
4. 商品情報を入力します。
5. 必要に応じて添付ファイルを追加します。
6. `一時保存` または `エントリー完了` を押します。

#### 主な入力項目

- 作成者
- メールアドレス
- 電話番号
- タイトル
- 棚割名
- 案件
- 補足情報
- 商品名
- JANコード
- 商品画像
- リスク分類
- 特定成分

補足:

- 商品名入力時、同名商品の過去データがあれば反映確認が表示されます。
- `過去商品検索` により過去登録商品を検索して流用できます。

【画面キャプチャ: エントリーシート一覧】

【画面キャプチャ: エントリーシート 基本情報】

【画面キャプチャ: 商品情報入力エリア】

### 6-3. エントリーシート修正

1. 一覧から対象シートを開きます。
2. 内容を修正します。
3. `一時保存` または `エントリー完了` を押します。

### 6-4. クリエイティブ確認

1. ステータスが `確認待ち` のシートを開きます。
2. `紐づくクリエイティブ` を確認します。
3. シート内容と相違がないか確認します。

【画面キャプチャ: 紐づくクリエイティブ表示エリア】

### 6-5. 承認

1. `確認待ち` のシートを開きます。
2. 問題がなければ `承認する` を押します。

結果:

- ステータスが `承認済み` になります。

【画面キャプチャ: 承認ボタン表示状態】

### 6-6. 差し戻し

1. `確認待ち` のシートを開きます。
2. `差し戻す` を押します。
3. 差し戻し理由を入力します。
4. `差し戻しを確定` を押します。

結果:

- ステータスが `差し戻し` になります。
- 差し戻し理由が記録されます。

【画面キャプチャ: 差し戻し理由入力欄】

## 7. 入力時の注意事項

### 7-1. エントリー完了時の主な必須チェック

- 作成者
- メールアドレス
- 電話番号
- タイトル
- 商品名
- JANコード

販促物ありの場合は、以下も必要です。

- 販促物幅
- 販促物画像

### 7-2. JANコードの条件

- 8桁、13桁、16桁のいずれか
- 半角数字のみ

### 7-3. 商品画像の条件

- 25MB以下
- 短辺1000px以上

### 7-4. エントリー完了（画像なし）になる条件

商品画像が未登録の商品がある状態で `エントリー完了` すると、ステータスは `エントリー完了（画像なし）` になります。

## 8. イレギュラーケース

### 8-1. 同時更新が発生した場合

他のユーザーが先に更新した場合、競合メッセージが表示されることがあります。最新内容を確認してから再保存してください。

### 8-2. JAN重複が発生した場合

JAN の重複が検知された場合は、既存データとの整合を確認してください。

### 8-3. 画像サイズが大きすぎる場合

25MBを超える画像はアップロードできません。必要に応じて画像サイズを調整してください。

### 8-4. クリエイティブ未紐づけで確認依頼したい場合

クリエイティブが紐づいていない状態では確認依頼できません。先に Admin がクリエイティブを登録または紐づけしてください。

### 8-5. 差し戻し理由が未入力の場合

差し戻し理由が未入力では、差し戻し確定できません。

### 8-6. 一般ユーザーが他メーカーを操作したい場合

一般ユーザーは自社メーカー以外のシート、アカウントを操作できません。

## 9. 仕上げ時の使い方

このマニュアルを最終版にする際は、以下の順で整えると実運用しやすくなります。

1. `【画面キャプチャ】` の位置に実画面を貼る
2. 不要な補足や社内向けメモを削る
3. 会社名、問い合わせ先、改訂日を表紙または末尾へ追記する
